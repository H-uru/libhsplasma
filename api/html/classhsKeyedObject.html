Detailed Description

The base class for any top-level PRP object.

Any class that has a unique plKey and can be loaded from a PRP is derived from this class. hsKeyedObject and plKey go hand-in-hand, and as such, each contains a reference to the other. For Creatable Class IDs, hsKeyedObject classes will always be in the < 0x0200 range, whereas non-keyed classes are >= 0x0200.

This must be overloaded by subclasses to parse PRC data specific to the class from the PRC document. It is called once for each tag that is a child to the Creatable tag itself, so subclasses should check for each supported tag and handle it accordingly if it matches. If the tag is not recognized, it should be passed along to the parent class.

◆ orderAfter()

virtual bool hsKeyedObject::orderAfter ( const hsKeyedObject *  ) const
virtual

Determine if this object's key should be ordered after the specified object's key (for use in plKey list sorting). Important Note: This is not a strict ordering – that is, if key A is not ordered after key B, it does not necessarily imply that key B should be ordered after key A.
